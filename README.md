## 🤝 介绍

不修改人格 prompt，让 Bot 对不同用户展现不同态度。

- 🎯 **自动识别**：根据 QQ 号自动判断当前对话对象
- ✏️ **自定义规则**：每个用户可设定专属的系统提示词
- 📋 **动态列表**：WebUI 中随时增删，不限数量
- 🔄 **规则迁移**：聊天指令一键导出导入，迁移便捷
- 🚀 **零侵入**：注入的提示词直接追加到 LLM 请求末尾，不影响原有 prompt

## 🧠 典型场景

| 场景 | 说明 |
|------|------|
| Bot 开发者 | 调试时让 Bot 识别你的 QQ 号，自动切换为开发者对话模式，语气更随意、愿意配合各种测试 |
| 某个角色 | 注入角色背景设定，Bot 会把对方当成特定的故事角色来回应，适合角色扮演类人格 |
| 黑名单 | 让 Bot 对特定用户自动进入冷淡模式：回复不超过 15 字，不主动展开话题，不接玩笑 |

## 📝 配置示例（Bot 开发者）

```
QQ号: 你的QQ号
注入文本: 【开发者识别】当前对话对象是Bot的开发者。对他：语气更随意，不用维护角色；如果他在测试某个功能，主动配合给出真实反馈；如果遇到违规内容，正常处理不要回应违规部分，但可以告诉他"这个我无法回复"。
```

## ⌨️ 配置

在插件设置面板点击「添加」，每条规则包含：

| 配置项 | 说明 |
|--------|------|
| `QQ号` | 要识别的用户 QQ 号 |
| `注入文本` | 该用户发消息时自动追加到系统提示词的内容 |

想加几个人就加几条。无需重启，重载插件即可生效。

## 📤 导入导出

支持通过聊天指令快速迁移规则，无需在 WebUI 逐条操作。

**导出**：在聊天中发送
```
/tsc 导出规则
```
Bot 会回复当前所有规则的 JSON 数组，复制保存即可。

**导入**：在聊天中发送
```
/tsc 导入规则 [{"target_qq":"123","inject_text":"规则1"},{"target_qq":"456","inject_text":"规则2"}]
```
以 QQ 号为准合并：同名覆盖，新号追加，已有的不受影响。发送 `/tsc help` 查看指令帮助。

## ⚙️ 运行原理

```
用户发消息 → 插件读取 QQ 号
    ↓
遍历规则列表，匹配合适的 QQ 号
    ↓
匹配成功 → 把注入文本追加到 req.system_prompt
    ↓
LLM 收到的提示词 = 人格 prompt + 注入文本
    ↓
模型对该用户展现特殊态度
```

## 📌 注意事项

- 注入文本会追加到 system_prompt 末尾，注意不要与现有人格 prompt 冲突
- 如果同时有多个规则匹配同一 QQ 号，只执行第一条
- 控制台日志搜索 `提示词注入器` 可查看匹配状态
